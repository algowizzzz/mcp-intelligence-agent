## Defect Summary

### AP-04 — No Upload Queue / Drop Zone
**Severity:** Medium
The UAT spec expects a drag-to-upload zone with per-file progress bars and success/error badges. The implementation uses a hidden `<input type="file" multiple>` triggered by the "↑ Upload" button. Files are uploaded sequentially via `fetch()` with only a toast notification per file. No drop zone for external file uploads exists; the drag/drop handlers in the DOM are for internal tree-to-folder moves only.

### AP-05 — Inline Rename Not Implemented
**Severity:** Low
The `.tree-rename-input` CSS class is defined (suggesting the feature was planned), but no `dblclick` event listener is attached to tree items. Rename is only accessible via the "rename" button in the hover-visible `.item-actions` panel, which triggers a browser `prompt()` dialog instead of an inline input field.

### AP-06 — No Right-Click Context Menu
**Severity:** Low
No `contextmenu` event listener exists in `admin.html`. Right-clicking any file or folder shows the browser's native context menu. File actions (Preview, Rename, Delete) are accessible via the hover-activated `.item-actions` button row on each tree item, but not via right-click.

### AP-13 — Audit Log Pagination Not Implemented in UI
**Severity:** Medium
The `/api/super/audit` endpoint fully supports `offset`, `limit`, `total_matched`, and `total_returned` response fields. However, `loadAudit()` in `admin.html` hardcodes `?limit=200` with no offset parameter and no pagination controls (next/prev buttons, page number display, or total_matched count). Users cannot page through more than 200 audit entries.
